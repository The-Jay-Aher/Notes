# Networking Fundamentals

## Quick Summary

Networking fundamentals explain how devices communicate, how IP addressing works, and how to troubleshoot connectivity. These notes are required before cloud networking, Kubernetes networking, load balancers, firewalls, DNS, and DevOps troubleshooting.

## Learning Order

| Order | Note | Focus |
| --- | --- | --- |
| 1 | [Fundamentals of Networking](1%20-%20Fundamentals%20of%20Networking.md) | OSI/TCP-IP models, switching, routing, DNS, DHCP, NAT, firewalls, ports, TCP/UDP, HTTP, troubleshooting. |
| 2 | [Internet Protocol (IP)](2%20-%20Internet%20Protocol%20%28IP%29.md) | IPv4, IPv6, CIDR, private/public IPs, routing, ARP, ICMP, NAT, cloud and Kubernetes IP concepts. |

## Revision Path

1. Understand the difference between switch, router, DNS, DHCP, NAT, firewall, and load balancer.
2. Learn how a request moves from application to transport to IP to link layer.
3. Practice IP/CIDR notation.
4. Practice troubleshooting with DNS, route, port, logs, and firewall checks.
